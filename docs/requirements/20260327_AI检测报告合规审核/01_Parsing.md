## 3. 业务流程与场景 (Process & Scenarios)

**场景 A0：认证经理端到端（与「识别 / 比对 / 打标」对齐）**

1. **触发：** 新报告入库（新测、年审、扩型号、换实验室等）。  
2. **Step 1 识别：** OCR/版式解析 + 字段抽取（产品信息、标准、等级、测试项、实测、限值、结论、附图引用等）。  
3. **Step 2 比对：** 同表内实测 vs 限值；结论 vs 数据；可选：与证书列名、与历史同 SKU 报告差异。  
4. **Step 3 打标：** 生成结构化问题单（类型、位置、严重度建议、原文片段/页码）。  
5. **输出：** 供认证经理复核、退回实验室或归档至 SKU/项目。

**场景 A：结构化数据与结论一致性审核**

1. **触发：** 用户提交 EMC / 安规 / 射频检测报告（或解析后的结构化字段）。
2. **前置依赖：** 报告中需能识别**实测值**、**限值**、**单位**、**适用标准与等级**（如 Class B）、**单项结论/总结论**。
3. **Step 1：** 抽取各频点或各测试项的实测值与对应限值。
4. **Step 2：** 校验 **实测值 ≤ 限值**（同一物理量、单位统一后再比）。
5. **Step 3：** 校验 **结论与数据一致**（若实测已超限，结论不得为 Pass）。
6. **Step 4：** 若违反规则，系统标红并输出明确提示（示例见下「领域细节锚点」）。

**场景 B：报告完整性 / 合理性辅助**

1. 关注 **EMC：** 辐射发射/传导骚扰是否可能超限；抗扰度等级是否达标；是否缺**测试布置图**；限值标准选用是否适用（如 **CISPR 32 vs CISPR 35**）。
2. 关注 **射频：** 发射功率、占用带宽、杂散发射、频率容限是否符合 **RED** 或 **FCC Part 15**；**SAR** 是否在限值内。
3. **触发：** 「数据可参考……看合理性」——偏专家规则 + 参照标准，不仅是一条数值不等式。

**场景 C：产品图（当前排除或待定）**

1. 潘小朋明确表示产品图「不是很好用 AI 审核」「细节太多」——**当前不作为自动审核范围**或需另案设计。

## 4. 领域细节锚点 (Domain Anchors)

### 关键数据（报告内字段 + 与跨境业务的衔接点）

- **报告内核心字段：** 实测值、限值、频率、功率、占用带宽、杂散、SAR、测试布置图有无、标准编号与 **Class** 等。  
- **工贸一体 / 跨境语境（待与业务确认是否纳入一期）：** 报告与 **SKU / 型号 / 目的国法规清单 / 平台类目要求** 的关联键（料号、ASIN、店铺站点）；用于「同 SKU 多版本报告」比对与上架前拦截。  
- **ERP 侧（隐性）：** 若认证结果需回写主数据或阻塞发货/上架，需定义**数据口径**与**状态机**（如：仅提示 / 强拦截），本资料尚未展开。

### 检测报告 / 合规类

| 类别 | 锚点 |
|------|------|
| **判定规则** | 测试数据**不得超过**报告中的限定值；**实测值 ≤ 限值**；结论须与数据一致。 |
| **单位与口径** | **单位必须统一**后方可比较。 |
| **标准 / 等级** | EMC：辐射/传导骚扰、抗扰度；限值标准适用性 **CISPR 32 vs CISPR 35**；示例限值语境 **EN 55032 Class B**。射频：**RED**、**FCC Part 15**；**SAR** 限值。 |
| **数值示例（对话中明确给出）** | 某频点实测 **42 dBμV**，限值 **40 dBμV**（EN 55032 Class B），若结论写 **Pass**，则属**数据超标但结论未反映**。 |
| **工具 / 产品名** | **IACheck**（示例）：应自动**标红**并提示类似「**数据超标但结论未反映**」——与「问题打标」能力同构。 |
| **打标类型（归纳自 IM，可扩展为标签体系）** | 数据超标、结论与数据不一致、单位/口径不一致、缺测试布置图、标准适用性疑问（如 CISPR 32/35）、射频指标类（功率/带宽/杂散/容限/SAR）等。 |
| **明确排除 / 边界** | **产品图片**：细节多，**暂不宜**作为 AI 审核主场景（仍在思考方案）。 |

### 业务限制（原文逻辑）

- **不能：** 在实测已超限时仍接受「Pass」类结论而不报警（产品期望行为）。
- **必须：** 单位统一；结论与数据一致；数据相对限值可判定。
